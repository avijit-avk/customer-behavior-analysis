## 📌 Overview

This is an end-to-end data analytics project that analyses customer shopping behaviour using a real-world retail dataset. The project covers the complete data analytics workflow — from raw data ingestion and cleaning, to SQL-based analysis, interactive dashboards, and a final presentation.

The goal is to uncover insights into customer demographics, purchase patterns, product performance, subscription behaviour, and revenue trends.

## 🔄 Project Workflow

### Step 1 — Load & Explore Data (Python)
- Loaded the CSV dataset using `pandas`
- Performed **Exploratory Data Analysis (EDA)**:
  - Checked data shape, column types, and value distributions
  - Identified missing/null values
  - Explored key statistics with `.describe()` and `.value_counts()`

### Step 2 — Clean Data (Python)
- Handled missing values using `df.replace()` and `np.nan`
- Standardised column formats
- Verified data integrity before export

### Step 3 — Export to MySQL (SQLAlchemy)
- Connected Python to MySQL using **SQLAlchemy + PyMySQL**
- Exported the cleaned DataFrame directly to MySQL using `df.to_sql()`

```python
from sqlalchemy import create_engine

engine = create_engine('mysql+pymysql://root:password@localhost:3306/customer_shopping_behaviour')

df.to_sql(name='customer_shopping_behaviour', con=engine, if_exists='replace', index=False)
```

### Step 4 — SQL Analysis (MySQL Workbench)
Wrote and executed **10 business-focused SQL queries** (see [SQL Queries](#-sql-queries) section below).

### Step 5 — Power BI Dashboard
- Connected Power BI directly to MySQL database
- Built an interactive **Customer Behavior Dashboard**
- Visualised KPIs, subscription breakdown, revenue by segment, and more

### Step 6 — Report & Presentation (Gamma)
- Created a professional report summarising findings
- Built a presentation using **Gamma AI**

---

## 📈 Key Findings

- **73%** of customers are not subscribed — a major upsell opportunity
- Subscribed customers have a **higher average spend** than non-subscribers
- **Clothing** is the top-performing category by number of purchases
- Customers with **more than 5 previous purchases** are more likely to subscribe
- **Standard and Express shipping** show minimal difference in avg purchase amount

## 🚀 How to Run This Project

### Prerequisites
```
Python 3.x
MySQL Workbench
Power BI Desktop
MySQL Connector/NET (for Power BI connection)
```

### 1. Clone / Download the project files

### 2. Run the Jupyter Notebook
```bash
# Install dependencies
pip install pandas numpy sqlalchemy pymysql

# Open the notebook
jupyter notebook customer_shopping_behaviour.ipynb
```

### 3. Export to MySQL
- Make sure MySQL is running locally
- Update credentials in the notebook
- Run the `df.to_sql()` cell to load data into MySQL

### 4. Run SQL Queries
- Open `Da-project-1SQL.sql` in MySQL Workbench
- Select your database and execute queries

### 5. Open Power BI Dashboard
- Open Power BI Desktop
- Connect to MySQL: `Home → Get Data → MySQL`
- Enter `localhost` / `customer_shopping_behaviour`
- Load the table and refresh visuals
